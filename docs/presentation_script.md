## 一、开场介绍（0:00–0:30）

**【画面：项目标题页 / 登录界面】**

> 大家好，我们的课程设计项目是"校园二手教材与物品交易平台"。
>
> 这个系统面向在校大学生，解决校园内二手教材、数码产品、生活用品等闲置物品的流通问题。系统分为普通用户端和管理员后台两个角色，覆盖了从商品发布、审核、浏览、购买到评价的完整交易闭环，并集成了 AI 智能辅助能力。

---

## 二、技术架构（0:30–1:15）

**【画面：项目结构图 / 代码目录截图 / 架构示意图】**

> 技术栈方面，我们使用 Java 8 作为开发语言，JavaFX 构建桌面端图形界面，Maven 进行项目构建管理，SQLite 作为本地轻量级数据库。AI 能力通过 OpenAI 兼容接口接入，支持 DeepSeek 等模型。
>
> 架构上采用经典的 MVC + Service + DAO 分层结构：
> - Controller 层负责接收页面事件和界面更新
> - Service 层封装核心业务逻辑和规则校验
> - DAO 层负责数据库 CRUD 操作
> - Util 层提供数据库连接、会话管理、密码哈希等公共能力
>
> 数据库包含 users、goods、goods_images、orders、cart_items、reviews、admin_logs 共 7 张核心表。系统首次启动时自动建表并初始化演示数据，做到开箱即用。

## 三、用户端功能演示（1:15–3:05）

### 3.1 注册与登录（1:15–1:30）

**【画面：注册页面 → 登录页面】**

> 首先演示用户注册登录。输入用户名、密码和联系方式即可完成注册。系统会校验用户名唯一性，被封禁的账号无法登录并给出明确提示。

### 3.2 商品浏览与搜索（1:30–1:50）

**【画面：用户首页，展示商品列表、搜索框、分类筛选、排序】**

> 登录后进入用户首页，可以看到所有审核通过的在售商品。支持关键词搜索、按分类筛选（教材、数码、服饰、生活用品、其他），以及按价格、新旧程度、发布时间排序。商品卡片展示主图、名称、价格等关键信息。

### 3.3 商品发布与 AI 描述优化（1:50–2:10）

**【画面：发布商品页面，填写信息、上传图片、点击 AI 优化】**

> 用户可以发布闲置商品，填写名称、分类、价格、新旧程度和描述，支持上传 0 到 3 张商品图片并实时预览。这里有一个亮点功能——点击"AI 优化描述"，系统会调用大语言模型对商品描述进行润色和补充，帮助卖家写出更吸引人的描述。发布后商品进入待审核状态。

### 3.4 商品购买与购物车（2:10–2:30）

**【画面：商品详情页 → 购买 → 购物车页面 → 批量结算】**

> 在商品详情页可以查看完整信息、全部图片和历史评价。用户可以直接购买，也可以加入购物车稍后批量结算。购物车支持勾选多个商品一键结算，系统会实时检查商品状态，已售出或已下架的商品无法结算。

### 3.5 个人中心（2:30–2:50）

**【画面：个人中心页面，切换"我的发布"、"我的订单"标签】**

> 进入个人中心，可以查看我发布的商品列表，对未售出的商品执行下架操作，下架后商品不再对其他用户展示。切换到订单页，可以查看我的购买订单历史和卖出订单记录，订单展示商品名称、成交价格、交易对象和下单时间。

### 3.6 评价与 AI 购买建议（2:50–3:05）

**【画面：订单评价 → AI 购买建议弹窗】**

> 对已完成的购买订单可以进行 1-5 星评价并填写评价内容。此外，在商品详情页还可以使用 AI 购买建议功能，系统会分析商品信息给出是否值得购买、价格是否合理等参考意见。

## 四、管理员端功能演示（3:05–4:15）

### 4.1 商品审核（3:05–3:20）

**【画面：管理员登录 → 商品审核页面】**

> 使用管理员账号登录后进入后台。商品审核页展示所有待审核商品，管理员可以查看商品详情和全部图片，选择审核通过或删除违规商品。所有操作自动记录到操作日志。

### 4.2 用户管理（3:20–3:35）

**【画面：用户管理页面，搜索、封禁操作】**

> 用户管理页展示全部注册用户，支持按用户名搜索。可以查看每个用户的发布商品数和成交数量，并执行封禁或解封操作。被封禁用户将无法登录系统。

### 4.3 数据统计与图表（3:35–4:00）

**【画面：统计页面，展示卡片、饼图、柱状图、折线图】**

> 数据统计页是管理员后台的核心。顶部展示总用户数、总商品数、今日成交量、总成交额等关键指标卡片。下方提供多种图形化统计：商品分类占比饼图、商品状态数量柱状图、最近 7 天成交量趋势折线图，以及详细统计数据表格。
>
> 这里还集成了 AI 运营分析和统计图解读功能，点击后 AI 会根据当前平台数据生成运营摘要和趋势解读，为管理决策提供参考。

### 4.4 操作日志（4:00–4:15）

**【画面：操作日志页面】**

> 操作日志页按时间倒序记录管理员的所有关键操作，包括审核商品、删除商品、封禁解封用户等，便于追溯和审计。

## 五、项目亮点与总结（4:15–4:50）

**【画面：总结页 / 架构图】**

> 总结一下本项目的几个亮点：
>
> **第一，完整的业务闭环。** 从注册、发布、审核、浏览、购买、购物车结算到评价，覆盖了二手交易的全流程。
>
> **第二，AI 智能辅助。** 集成了商品描述优化、购买建议、搜索辅助、运营分析、统计解读共 5 项 AI 能力。采用真实接口优先、本地 Mock 兜底的策略，确保无网络环境下也能正常演示。
>
> **第三，商品图片管理。** 支持本地上传 0-3 张图片，发布时预览删除，列表展示主图，详情页展示全部图片。
>
> **第四，清晰的分层架构。** MVC + Service + DAO 分层，职责明确，代码可维护性好。数据库访问参数化，密码哈希存储，注重安全性。
>
> **第五，开箱即用。** 首次启动自动建表、初始化演示数据，支持打包为绿色版直接运行。

## 录制建议

1. 提前运行 `mvn clean compile && mvn exec:java` 确保系统正常启动
2. 使用屏幕录制软件（如 OBS）录制桌面操作
3. 配合画外音朗读脚本，语速适中，每段之间自然过渡
4. 关键操作放慢速度，确保观众能看清界面变化
5. AI 功能演示时可配置真实 API Key 展示实际效果，或使用 Mock 模式展示稳定输出
6. 建议分辨率 1920×1080，字体放大确保录制清晰
